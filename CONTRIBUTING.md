# Contributing to PushFlow

Thank you for considering contributing to PushFlow! We welcome contributions from everyone.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/web-push-noti-PWA.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Run linting: `pnpm lint`
6. Format code: `pnpm format`
7. Commit with conventional commit format: `git commit -m "feat: add new feature"`
8. Push to your fork: `git push origin feature/your-feature-name`
9. Open a Pull Request

## Commit Convention

We use [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation changes
- `style:` - Code style changes (formatting, etc.)
- `refactor:` - Code refactoring
- `test:` - Adding or updating tests
- `chore:` - Maintenance tasks

## Code Style

- We use ESLint and Prettier for code quality
- Pre-commit hooks automatically check linting and formatting
- Follow existing code patterns and conventions

## Pull Request Guidelines

- Keep PRs focused on a single feature or fix
- Update documentation if needed
- Ensure all tests pass
- Add tests for new features
- Keep commits clean and atomic

## Need Help?

Open an issue for questions or discussions!
